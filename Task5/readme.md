## Модель изоляции данных между партнёрами

| Уровень изоляции | Реализация | Преимущества | Недостатки |
| --- | --- | --- | --- |
| База данных | Отдельная схема для каждого партнёра в единой БД | <br> Экономия ресурсов<br> Простое управление резервными копиями | <br> Риск утечки при ошибках в коде |
| Кэш (Redis) | Префикс ключей: tenant:{id}:key | <br> Общая инфраструктура<br> Простая очистка кэша партнёра | <br> Требует строгой дисциплины в коде |
| Очереди (Kafka) | Префикс топиков: tenant.{id}.events | <br> Изоляция потоков событий<br> Независимое масштабирование | <br> Увеличение количества топиков |
| Хранилище файлов | Отдельный bucket/prefix для партнёра | <br> Полная изоляция медиафайлов | <br> Сложность кросс-тенантного доступа |
| Прикладной уровень | Middleware TenantContextFilter | <br> Централизованная проверка доступа<br> Автоматическая фильтрация запросов | <br> Требует тестирования всех путей |

## Этапы онбординга

- Регистрация — партнёр заполняет форму на портале (название, домен, контактные данные)
- Валидация — проверка уникальности домена и контактных данных
- Создание инфраструктуры — автоматическое создание:
  - Схемы БД с именем tenant_{id}
  - Префиксов в Redis: tenant:{id}:*
  - Топиков Kafka: tenant.{id}.bookings
- Настройка IAM — регистрация клиента OAuth2 в Keycloak с уникальным client_id
- Активация — администратор партнёра получает приглашение по email и завершает настройку

## Разграничение доступа

- Разделение обязанностей: Администратор партнёра не может видеть данные других партнёров
- Минимальные привилегии: Водители и пассажиры видят только свои данные
- Аудит доступа: Все операции с данными логируются с указанием tenant_id и user_id
- Динамические политики: Роли могут быть расширены через конфигурацию без изменения кода

## Роли и уровни доступа

| Роль | Описание | Доступ к данным партнёра | Доступ к глобальным данным | Возможности |
| --- | --- | --- | --- | --- |
| Глобальный администратор | Управление платформой | Полный доступ ко всем партнёрам | Полный доступ | <br> Создание/удаление партнёров<br> Настройка глобальных политик<br> Мониторинг всех метрик |
| Администратор партнёра | Управление партнёрским аккаунтом | Полный доступ к своему партнёру | Нет доступа | <br> Управление пользователями партнёра<br> Настройка кастомизации<br> Просмотр дашборда партнёра |
| Менеджер партнёра | Операционное управление | Чтение/запись данных партнёра | Нет доступа | <br> Просмотр отчётов<br> Управление тарифами партнёра<br> Блокировка пользователей |
| Водитель партнёра | Исполнитель заказов | Только свои данные | Нет доступа | <br> Приём заказов<br> Обновление статуса поездки<br> Просмотр истории своих поездок |
| Пассажир партнёра | Заказчик услуг | Только свои данные | Нет доступа | <br> Создание заказов<br> Просмотр истории своих поездок<br> Оценка водителей |
| Аналитик партнёра | Анализ данных | Только агрегированные данные | Нет доступа | <br> Просмотр дашбордов<br> Экспорт отчётов (анонимизированных) |
| Поддержка партнёра | Техническая поддержка | Ограниченный доступ к данным | Нет доступа | <br> Просмотр деталей заказов (без ПД)<br> Общение с пользователями через систему |

## Система мониторинга мультитенантного окружения

### Ключевые метрики с разделением по партнёрам:

| Категория | Метрика | Порог тревоги | Описание |
| --- | --- | --- | --- |
| Изоляция данных | tenant.isolation_violations | > 0 | Попытки доступа к данным другого партнёра |
| Производительность | tenant.latency_p95 | > 2 сек | Задержка для партнёра |
| Нагрузка | tenant.requests_per_minute | > 1000 | Пиковое количество запросов |
| Ошибки | tenant.error_rate | > 1% | Доля ошибочных запросов |
| Хранилище | tenant.db_size_gb | > 100 ГБ | Объём данных партнёра |
| Пользователи | tenant.active_users | Резкий рост/падение | Активность пользователей партнёра |

## Дашборд партнёра в Grafana:
- Виджеты:
  - Количество активных заказов (сравнение с прошлым периодом)
  - Среднее время назначения водителя
  - Доля успешных платежей
  - Рейтинг водителей
- Фильтры:
  - Выбор периода (день/неделя/месяц)
  - Фильтр по региону (если партнёр работает в нескольких)
  - Сравнение с другими партнёрами (анонимизированное)
