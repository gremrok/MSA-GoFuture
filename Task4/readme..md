## Механизмы обеспечения качества данных

| Этап пайплайна | Механизм | Инструменты | Метрики качества |
| --- | --- | --- | --- |
| Сбор данных | Валидация схемы событий | JSON Schema + Confluent Schema Registry | <br> Процент валидных событий > 99.9%<br> Количество отклонённых событий в DLQ |
| Потоковая обработка | Мониторинг задержек и полноты | Flink Metrics + Prometheus | <br> Lag потребителя < 10 сек<br> Полнота данных (99.5% событий обработано) |
| Хранение | Проверка целостности и дедупликация | Great Expectations + dbt tests | <br> Отсутствие дубликатов<br> Проверка NULL-значений в критичных полях |
| ML-признаки | Мониторинг дрейфа и аномалий | Evidently AI + Custom Alerts | <br> Статистический дрейф (PSI < 0.1)<br> Аномалии в распределении признаков |
| BI-отчёты | Тестирование бизнес-логики | dbt tests + Dataform | <br> Согласованность агрегатов между источниками<br> Отклонение от ожидаемых значений < 5% |

Система мониторинга и сбора данных для GoFuture построена как единая платформа, объединяющая операционную наблюдаемость (трейсы, метрики, логи) и аналитику для бизнеса и ML. 

Все новые компоненты (выделены красным) интегрируются с существующими микросервисами через стандартные протоколы (Kafka, OTLP), минимизируя изменения в коде. 

Механизмы обеспечения качества данных (валидация схем, мониторинг полноты, отслеживание дрейфа) гарантируют надёжность данных для критически важных ML-моделей динамического ценообразования и обнаружения фрода. 

Платформа поддерживает полный цикл: от сбора событий в реальном времени до визуализации в BI-инструментах и инференса моделей.
